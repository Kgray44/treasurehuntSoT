# Action risk model

- Low: navigation, refresh, preview, reconciliation request.
- Medium: preparation and staging; clear acknowledgement, no typed phrase.
- High: player-visible release, solve/complete, pause/resume, award, reveal, and undo; server projection and confirmation required.
- Critical: force progression, multi-event revert, reset, finale unlock; not exposed in Phase 3.

Confirmation shows command, event, current/projected sequence, affected systems, ceremony, undo availability, and unmet prerequisites. A prerequisite failure disables commit and is announced without relying on color.
