# Phase B-6 external usability report

## Result

External participant count: **0**. No outside Creator, uninvolved Player, or uninvolved Captain completed the required study. The implementation author and automated browser tests do not count as external validation. The usability gate fails and B6-008 remains open.

## Prepared study

Recruit at minimum: one technically comfortable outside Creator, one non-technical Creator, and one accessibility-relevant participant; add uninvolved Player and Captain roles. Do not coach beyond the public guides.

Creator scenario: install/launch, orient, create a waypoint, record target/accepted/boundary/two negatives, edit stable/ignored regions, correct a health warning, build/test, interpret reliability, publish a new immutable version, bind it to a story, and explain privacy/versioning.

Player scenario: pair/select, identify Vision Active, hold-to-scan, respond to insufficient/ambiguous guidance, pause/stop, and explain processed/retained data.

Captain scenario: inspect exact versions/gates, approve/reject/request rescan, perform one manual recovery, truth-label a case, and explain audit/privacy effects.

Record completion, time, errors, interventions, abandonment, confidence, System Usability Scale or equivalent, keyboard/focus/reduced-motion issues, misleading language, privacy comprehension, and whether docs—not developer help—resolved problems. Any release-blocking finding must be fixed and rerun with fresh participants.
