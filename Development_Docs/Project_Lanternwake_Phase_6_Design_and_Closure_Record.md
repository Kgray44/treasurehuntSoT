## Non-negotiable completion rule

An accepted requirement closes only when it is reachable on its production surface, has the declared runtime and real trigger, preserves authoritative state, provides full/gentle/product-reduced/browser-reduced behavior plus a static failure fallback, and has deterministic test evidence. A development-showcase result, a fallback alone, or an architecture contract alone is not a closure result.

## Frozen implementation policy

1. **Runtime ownership.** GSAP owns narrative sequencing and SVG/path effects; Motion owns React presence/layout/interaction; Rive owns stateful authored relics; Lottie owns contained ambient or semantic effects; StPageFlip owns curl; CSS/WAAPI own bounded material micro-effects; Web Audio is optional reinforcement only. No runtime may claim an already-owned property.
2. **Motion modes.** Full mode may use the authored journey. Gentle mode retains sequence and meaning with shorter, lower-distance choreography. Product-, browser-, and combined-reduced modes use stable semantic poses with no nonessential travel. A mode change must not consume a one-shot or replay business state.
3. **Accessibility.** Keyboard and focus behavior are equivalent to pointer interaction; focus never enters hidden PageFlip source content; status text and live announcements carry meaning without sound; every failure fallback remains actionable at 200% zoom.
4. **Lifecycle.** Scenes are interruptible and abortable. Replays are presentation-only. Unmount, visibility change, route transition, mode change, offline failure, and repeated invocation return ownership, listeners, runtime instances, and timers to baseline.
5. **Performance.** Animate compositor-friendly properties where possible, pause ambient work while hidden, cap mobile ambient work, keep one dominant movement focus, and record timing/long-task/runtime-count evidence before final acceptance.
6. **Assets and provenance.** A production Rive entry is runtime-ready only with a project-owned authoring source/export pair, verified artboard/state machine/input schema/reduced pose/static fallback, SHA-256, local reachability, and license/provenance. SVG fallbacks and the MIT development sample remain distinct and may not be renamed or represented as production Rive.
